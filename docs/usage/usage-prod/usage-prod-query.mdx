# 数据使用

在数据上传完成后，您就可以自由使用数据。

## Dashboard（施工中）

我们内置了一个dashboard来简化一些通用操作。他目前功能还很少，但不失为一个合适的引语。你可以通过命令直接启动它：

```bash
./sibyl frontend
```

启动后打开 `localhost:3000` 即可：

![](https://user-images.githubusercontent.com/13421694/216641341-c01bbcd1-349f-4934-bd35-2fa6b2c48cb4.png)

如无意外，在顶部您会看到上一步上传的应用信息。选择指定的版本之后即可正常使用其他功能了。

:::tip
- 虽然这个dashboard可以搞出很多很酷炫的功能，但目前我们的工作重心暂时在核心功能与接口开发上。
- 如果您对前端感兴趣，欢迎帮助我们改进。dashboard 基于官方的 javascript client 开发，它并不复杂。
- [dashboard](https://github.com/opensibyl/sibyl-frontend) 有独立的仓库，sibyl2将会自动集成主分支上的内容，您无需担心分发问题。
:::

## 数据结构

逻辑上的存储接口是树形的：

- repo（应用
    - rev1（版本
        - file（文件
            - function
            - function ctx
            - class
    - rev2
        - file
            - function
            - function ctx
            - class

一般来说，你需要：

- 先查询你要选择的仓库
- 查询其名下的仓库
- 再带着他们的信息去查询他们名下的其他信息

## http 接口

你可以直接打开 http://127.0.0.1:9876/ops/swagger/index.html#/ 查看接口文档。

![](/img/usage-swagger.png)

通常你只需要关注五个接口：

- /repo：用于查询当前一共有哪些代码仓
- /rev：用于查询指定代码仓名下的版本
- /file：查询特定仓特定版本下的文件列表
- /func：函数信息查询
- /funcctx：函数信息及上下文查询

## sdk

可以利用sdk与你的应用轻松集成：

- golang: https://github.com/opensibyl/sibyl-go-client
- java: https://github.com/opensibyl/sibyl-java-client

但我们的 SDK 均使用 openapi-generator 自动生成，在设计上可能有些反直觉。建议前往 [idea一节](../../idea/idea-intro) 结合实际例子理解。
